# AGENTS.md

## Project overview
- This is a beginner-friendly Java game project built with libGDX.
- The goal is to create an original top-down paint-action prototype.
- The long-term inspiration is a territory-control game, but do not copy names, characters, stages, UI, music, sounds, or branding from existing commercial games.

## Tech stack
- Language: Java
- Framework: libGDX
- Target platform: Desktop first
- Keep the architecture simple and beginner-friendly.

## Current scope
- Phase 1: Player movement
- Phase 2: Shooting
- Phase 3: Paint floor tiles when bullets hit
- Phase 4: Calculate painted area score
- No online multiplayer yet
- No advanced shaders yet
- No complex enemy AI yet

## Working rules
- Before editing code, explain the plan in Japanese.
- Read the existing files before making changes.
- Make small, reviewable changes.
- Do not add new dependencies unless necessary.
- Ask before adding a new library.
- Prefer simple class structures over advanced abstractions.
- Add comments for important logic so a beginner can follow the code.
- When you finish, list the changed files.

## Build and run
- Check README.md and build files first.
- If build or run commands are missing, propose them before editing.
- After changes, explain how to run the game.

## Definition of done
- The project builds successfully.
- The new feature works locally.
- Manual test steps are written in simple Japanese.
- Known limitations are clearly listed.
- The explanation is understandable for a beginner.
